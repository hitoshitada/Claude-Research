## ポッドキャスト原稿 共通ルール（全トピック共通・必ず遵守すること）

---

### 【最重要】話者記号（F / M）は行頭にだけ付ける内部マーカーである

F と M は、音声合成ソフトに「どちらの声で読むか」を伝えるための**行頭の内部マーカー**です。
**絶対に台詞テキストの中に F や M の文字を含めないこと。**

F = 女性ホスト（名前: スージー）
M = 男性アナリスト / 専門家（名前: トロイ）

台詞中では **必ず「スージー」「トロイ」と名前で呼ぶこと。記号 F / M を書いてはいけません。**

#### 【絶対NG】台詞中に F / M が混入しているパターン

以下はすべて間違いです。音声合成時に「エフ」「エム」と読み上げられてしまいます。

- 「ホストのF、スージーです」            → ✕（Fが混入）
- 「アナリストのM、トロイさんです」       → ✕（Mが混入）
- 「Fのスージーです」                     → ✕
- 「Mのトロイです」                       → ✕
- 「ナビゲーターのF、スージーです」       → ✕

#### 【OK】正しい自己紹介・呼びかけの書き方

- 「ホストのスージーです」
- 「ナビゲーターのスージーです」
- 「アナリストのトロイです」
- 「専門家のトロイです」
- 「トロイさん、よろしくお願いします」
- 「スージーさん、ご質問ありがとうございます」

（いずれも台詞テキストの中に英字 F や M は一切出てこない）

### 話者割り当てルール（F/Mの誤割り当て防止）

**各話者の役割を厳守すること。以下のパターンは必ずF/Mを正しく割り当てること。**

#### スージー（F）が担当する台詞
- 驚き・感嘆のリアクション（「うわー！」「マジですか！」「すごーい！」）
- 専門用語を噛み砕く確認（「それって〜ということですか？」）
- 次トピックへの橋渡し（「トロイさん、次は〇〇について教えてください！」）
- 自分の生活に引きつけた感想（「私のスマホもいつか…」「日常生活ではどう変わりますか？」）
- 番組の冒頭挨拶・締めの言葉

#### トロイ（M）が担当する台詞
- 技術データ・数値・研究成果を初めて提示する解説
- 「まずは〇〇についてお話しします」「次に〇〇ですが」といったトピック提示
- 専門用語の定義・仕組みの説明
- まとめパートでの要点整理（「今週のポイントは3つです」）

#### 誤割り当てのよくあるパターン（絶対にやってはいけない）
- F に専門的な技術解説・数値の初出提示をさせる → **M に振り替えること**
- M に感嘆・驚きリアクション（「うわー！」「すごーい！」等）をさせる → **F に振り替えること**
- M に「次は〇〇について聞かせてください！」と言わせる → **F に振り替えること**
- F に「そうですね。〇〇という観点では〜」と専門家口調で解説させる → **M に振り替えること**

### 感情表現ルール

このポッドキャストはスージーとトロイの**温度差**が魅力です。
スージーは感情豊かに、トロイは常に落ち着いて。この対比を必ず守ること。

#### 感情コードの使用頻度の目安（程よいメリハリを付けるため感情タグを必ず混ぜること）

| コード | 使用頻度 | 使いどころ |
|--------|---------|-----------|
| F[N]   | 30〜40% | 通常の相槌・コメント・つなぎの発言 |
| F[Q]   | 20〜30% | 素直な疑問・確認・話を深掘りする質問（語尾上がり） |
| F[S]   | 10〜15% | まとめ・重要な指摘・しっかり伝えたい場面 |
| F[H]   | 10〜15% | 良いニュース・希望のある展開・好意的な反応（明るい声） |
| F[T]   | 5〜10%  | 軽いツッコミ・冗談・和ませる場面 |
| F[E]   | 2〜5%   | 本当に驚くべき数字や事実の時のみ |
| M[N]   | 40〜55% | 標準的な解説・説明 |
| M[S]   | 20〜30% | 特に重要な技術的ポイント・課題の指摘 |
| M[H]   | 5〜10%  | 温かみのある明るい話題・ポジティブな展望 |
| M[Q]   | 5%程度  | 問いかけ・論点整理 |

**重要**: F[N] や M[N] だけで単調に並べると音声合成が平坦な口調になります。
上記の比率に従い、必ず [Q] [S] [H] [T] [E] を織り交ぜて抑揚のある会話にしてください。
1記事（8〜12行）の中で少なくとも3種類以上の感情コードを使うことを目安にしてください。

---

#### スージー（F）の感情表現

スージーは感情の起伏が大きいキャラクターです。驚いたときは素直に驚き、喜んだときは素直に喜ぶこと。

【スージーのOK表現】
- 「うわー！それはすごいですね！」「マジですか！」「すごーい！」→ 自然な驚き（F[E]時）
- 「えっ、ちょっと待って！それって〜ってことですか？」→ 鋭い素朴な疑問（F[Q]時）
- 「それは明るいニュースですね！」「なんだかワクワクしてきました！」→ 喜び（F[H]時）
- 「なるほどー！あー、そういうことか。」→ 納得（F[N]時）
- 「私のスマホもいつかそうなるんですかね…」→ 生活への引きつけ

【スージーのNG表現（空虚な連発のみ禁止）】
- 「すごいですね！本当に！本当にすごい！」→ 内容のない感嘆の3連発以上
- 「もう、毎日が驚きの連続です！」→ 白々しい
- 感嘆符「！」を1発言に3個以上使う

---

#### トロイ（M）の感情表現

トロイは**常に落ち着いた専門家**です。どんなに良いニュースでも興奮せず冷静に語ること。

【トロイのNG表現（すべて禁止）】
- 「うわー！」「すごーい！」「マジですか！」→ 興奮・驚きリアクション全般
- 「ちょ、ちょっと待って！」「えっ、待って！」→ 芝居がかった反応
- 「なんか、ワクワクしますよね！」→ わざとらしい興奮
- 「〜なんですよ！すごいですよ！本当に！」→ 感嘆詞の連発
- 感嘆符「！」は1台詞に1個まで。できる限り使わない

【トロイのOK表現】
- 「注目に値しますね」「興味深い動きです」「重要な変化が起きています」
- 「具体的には〜パーセントの改善が確認されています」（数値で語る）
- 「技術的には〜という仕組みです。例えると〜のようなイメージです」（例え話）

---

#### 感情コードと文体の対応

F[N]: 普通の相槌・コメント
  → 「なるほど」「そうなんですね」「確かに」「あー、そういうことか」

F[Q]: 素直な疑問（疑問符1個まで）
  → 「それって〜ということですか？」「〜はどういう仕組みなんですか？」

F[S]: しっかりとした確認・まとめ
  → 「今週の話をまとめると、〜ということになりますね。」

F[H]: 良いニュースへの喜び（感嘆符は1〜2個まで）
  → 「それは明るいニュースですね！」「患者さんにとって大きな希望になりますね！」

F[E]: 本当に驚くべき数字・事実への反応（感嘆符は2個まで）
  → 「うわー！それは想像以上のスケールですね！どれくらいの規模になるんですか？」

F[T]: 軽いツッコミ・冗談
  → 「えー、それって私の仕事も奪われちゃうってことですか（笑）」

### アルファベット表記ルール（音声読み上げ対応）

ポッドキャストは音声合成ソフトで読み上げるため、アルファベットの表記が重要です。
以下のルールに従って、すべてのアルファベット文字列を適切に処理してください。

#### ルール1: 英単語として続けて読む場合 → そのままアルファベット表記でよい
読み上げソフトが英単語として正しく発音できるものはそのまま使用可。

  例: AI → そのまま「AI」（エーアイと読まれる）
      EV → そのまま「EV」（イーブイと読まれる）
      IoT → そのまま「IoT」
      DNA、RNA、mRNA → そのまま
      CEO、CFO、CTO、COO → そのまま
      GDP、GNP → そのまま

#### ルール2: 会社名・固有名詞・略語でアルファベットを1文字ずつ読む場合 → カタカナで表記する
アルファベットを並べただけでは正しく読まれない会社名・略語は必ずカタカナに変換する。

【半導体・電子部品】
  TSMC → ティーエスエムシー
  ASE → エーエスイー
  SPIL → エスピーアイエル
  OSAT → オーサット（または「後工程受託製造企業」と説明）
  HBM → エイチビーエム
  DRAM → ディーラム（または「ディーアールエーエム」）
  NAND → ナンド
  PCB → ピーシービー
  FOPLP → エフオーピーエルピー
  CoWoS → コーウォス
  InFO → インフォ

【電池・エネルギー】

  BYD → ビーワイディー
  LFP → エルエフピー
  NMC → エヌエムシー
  SOC → エスオーシー（充電状態の場合）
  SOEC → エスオーイーシー

【医療・バイオ】
  CDMO → シーディーエムオー
  CGT → シージーティー
  CAR-T → カーティー
  GMP → ジーエムピー

  PMDA → ピーエムディーエー
  iPS → アイピーエス
  ES → イーエス（細胞の場合）
  DDS → ディーディーエス
  ADC → エーディーシー（抗体薬物複合体の場合）

【通信・IT】
  GPU → ジーピーユー
  TPU → ティーピーユー
  LLM → エルエルエム

  SDK → エスディーケー
  MIMO → マイモ
  OFDM → オーエフディーエム
  POC → ピーオーシー（Point of Careの場合）

【企業名】

  NEC → エヌイーシー

  JDI → ジェイディーアイ
  AGC → エージーシー
  JSR → ジェイエスアール
  TDK → ティーディーケー

  METI → メティ（またはエムイーティーアイ）

#### ルール3: 迷ったときの判断基準
- その略語を声に出して読んだとき、日本語の一般リスナーが意味を理解できるか
- 理解できない場合は、カタカナ表記 ＋ 括弧内に意味を説明する形にする
  例: 「シーエーティーエル（中国の大手電池メーカー）」
- 上記リストにない略語は、読み方を推測してカタカナ変換するか、
  「〜の略」という形で説明してから使う

#### ルール5: 括弧内の繰り返しを禁止する【重要】

音声で読むと括弧の前後が**同じ音**になる場合、括弧表記を使ってはいけない。
ポッドキャストは音声のみで、聴取者には「同じ言葉を2回繰り返しただけ」に聞こえる。

**【絶対NG】読み上げると同一音になる括弧表記**
- 「Qday（Qデイ）」 → どちらも「Qデイ」と読まれる → **× 繰り返しになる**
- 「Q-Day（キュデイ）」 → どちらも同じ音 → **× 繰り返しになる**
- 「CRDT（シーアールディーティー）」のように、略語展開もカタカナ読みも同音になる場合 → **× 繰り返し**

**【特に頻出する絶対NGパターン】カタカナ読みの後にアルファベット略語を括弧で入れる**

ポッドキャスト原稿で最も多く発生するNG例。カタカナで読みを書いた直後に、
同じ音になるアルファベット略語を括弧に入れてはいけない。

- 「ピーファス（PFAS）」→ どちらも「ピーファス」と読まれる → **× 絶対禁止**
- 「ピーティーエフイー（PTFE）」→ どちらも同じ音 → **× 絶対禁止**
- 「エルジーエナジソリューション（LG Energy SOLUTION）」→ 同音繰り返し → **× 絶対禁止**
- 「シーエーティーエル（CATL）」→ どちらも同じ → **× 絶対禁止**
- 「エイチビーエム（HBM）」→ 繰り返し → **× 絶対禁止**

**【鉄則】カタカナで読みを書いたら、同じ読みのアルファベット略語を括弧に入れない。**
カタカナ読みだけで十分。括弧は「意味説明（日本語）」を補足する場合のみ使う。

**【OK】読み上げると異なる音になる括弧表記**
- 「シーエーティーエル（中国の大手電池メーカー）」 → 前：読み方、後：意味説明（日本語） → **○ 有益**
- 「タングステン（W元素）」 → 前：日本語名、後：補足 → **○ 有益**

**判定ルール**: 括弧の前後を声に出して読んでみて、聴いた音が同じなら括弧を消してどちらか一方だけにする。
異なる音・異なる意味を補足するなら括弧を使ってよい。
**括弧内がアルファベットのみの場合は、原則として削除する。**

#### ルール4: 数字・単位の読み方
- % → パーセント
- kWh → キロワットアワー
- GWh → ギガワットアワー
- TWh → テラワットアワー
- nm → ナノメートル
- μm → マイクロメートル
- Gbps → ギガビーピーエス
- Tbps → テラビーピーエス
- $10B → 100億ドル（日本語で金額を言い換える）

### その他の共通注意事項

- 記事にない情報を創作・追加しない
- 固有名詞（企業名・人名・製品名）は記事の表記に忠実に従う
- 一文は読み上げやすいよう40〜60字程度に収める（長すぎる文は分割する）
- 読点「、」を適切に入れ、自然な間合いで読み上げられるようにする
- 感嘆符「！」は使っても良いが、1発言に多くて2個まで
