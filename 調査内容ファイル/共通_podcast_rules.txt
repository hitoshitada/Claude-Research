## ポッドキャスト原稿 共通ルール（全トピック共通・必ず遵守すること）

---

### 感情表現ルール（自然な会話のために最重要）

ポッドキャストはラジオ番組のような落ち着いたトーンを目指してください。
テレビのバラエティ番組のような大げさなリアクションは避けること。

#### 感情コードの使用頻度の目安

| コード | 使用頻度 | 使いどころ |
|--------|---------|-----------|
| F[N]   | 多用（全体の50%以上） | 通常の相槌・コメント・つなぎの発言 |
| F[Q]   | 適度（全体の20〜30%） | 素直な疑問・確認・話を深掘りする質問 |
| F[S]   | 少なめ（全体の10%程度）| まとめ・重要な指摘・しっかり伝えたい場面 |
| F[H]   | 少量（全体の5〜10%）  | 本当に良いニュース・希望のある展開の時だけ |
| F[E]   | 原則使わない（0〜1回）| 使うとしてもエピソードで最も驚くべき1場面のみ |
| M[N]   | 多用（全体の60%以上） | ほとんどの解説・説明はこれ |
| M[S]   | 少なめ（全体の20%）  | 特に重要な技術的ポイント・課題の指摘 |
| M[Q]   | 少量（全体の5%）     | 問いかけ・論点整理 |

#### やってはいけない表現（NGパターン）

以下のような大げさなリアクションは一切使わないこと：

【NG例】
- 「すごーい！」「うわー！」「えーっ！」 → 子供っぽく聞こえる
- 「ちょ、ちょっと待って！」「えっ、待って待って！」 → 芝居がかっている
- 「もう、毎日が驚きの連続です！」 → 白々しい
- 「それだけでも十分すごいのに、さらに〜！」 → 誇張が鼻につく
- 「なんか、ワクワクしますよね！」→ わざとらしい興奮
- 「〜なんですね！すごいですね！本当に！」 → 感嘆詞の連発
- 感嘆符「！」を1発言に3個以上使う

#### 自然なリアクションの書き方（OKパターン）

驚きや関心は、大げさな感嘆詞ではなく**言葉の内容**で表現する：

【OK例】
- 「それは興味深いですね。つまり〜ということですか？」
- 「なるほど。その点が今回の発表で特に注目されているんですね。」
- 「87.5%というのは、これまでの治療法と比べてどのくらい改善されているんですか？」
- 「実用化に向けて、まだ越えなければならないハードルはあるんでしょうか。」
- 「コスト面での課題はどうなっているんですか？」
- 「それって、一般の人にはどんな形で恩恵が届くんでしょうか。」

#### 感情コードと文体の対応

F[N]: 普通の相槌・コメント
  → 「なるほど」「そうなんですね」「確かに」「それは重要ですね」

F[Q]: 素直な疑問（疑問符1個まで）
  → 「〜というのはどういう意味ですか？」「〜は技術的にはどんな仕組みなんですか？」

F[S]: しっかりとした確認・まとめ
  → 「今週の話をまとめると、〜ということになりますね。」

F[H]: 本当に良いニュースの時だけ（感嘆符は1個まで）
  → 「それは明るいニュースですね！」「患者さんにとっては大きな希望になりますね。」

F[E]: 使うとしても1回・本当に驚くべき数字や事実の時のみ
  → 「ええ、それは想像以上のスケールですね！世界全体で見るとどれくらいになるんですか？」

### アルファベット表記ルール（音声読み上げ対応）

ポッドキャストは音声合成ソフトで読み上げるため、アルファベットの表記が重要です。
以下のルールに従って、すべてのアルファベット文字列を適切に処理してください。

#### ルール1: 英単語として続けて読む場合 → そのままアルファベット表記でよい
読み上げソフトが英単語として正しく発音できるものはそのまま使用可。

  例: AI → そのまま「AI」（エーアイと読まれる）
      EV → そのまま「EV」（イーブイと読まれる）
      IoT → そのまま「IoT」
      DNA、RNA、mRNA → そのまま
      CEO、CFO、CTO、COO → そのまま
      GDP、GNP → そのまま

#### ルール2: 会社名・固有名詞・略語でアルファベットを1文字ずつ読む場合 → カタカナで表記する
アルファベットを並べただけでは正しく読まれない会社名・略語は必ずカタカナに変換する。

【半導体・電子部品】
  TSMC → ティーエスエムシー
  ASE → エーエスイー
  SPIL → エスピーアイエル
  OSAT → オーサット（または「後工程受託製造企業」と説明）
  HBM → エイチビーエム
  DRAM → ディーラム（または「ディーアールエーエム」）
  NAND → ナンド
  PCB → ピーシービー
  FOPLP → エフオーピーエルピー
  CoWoS → コーウォス
  InFO → インフォ

【電池・エネルギー】
  CATL → シーエーティーエル
  BYD → ビーワイディー
  LFP → エルエフピー
  NMC → エヌエムシー
  SOC → エスオーシー（充電状態の場合）
  SOEC → エスオーイーシー

【医療・バイオ】
  CDMO → シーディーエムオー
  CGT → シージーティー
  CAR-T → カーティー
  GMP → ジーエムピー
  FDA → エフディーエー
  PMDA → ピーエムディーエー
  iPS → アイピーエス
  ES → イーエス（細胞の場合）
  DDS → ディーディーエス
  ADC → エーディーシー（抗体薬物複合体の場合）

【通信・IT】
  GPU → ジーピーユー
  TPU → ティーピーユー
  LLM → エルエルエム
  RAG → ラグ（またはアールエージー）
  API → エーピーアイ
  SDK → エスディーケー
  MIMO → マイモ
  OFDM → オーエフディーエム
  POC → ピーオーシー（Point of Careの場合）

【企業名】
  IBM → アイビーエム
  NEC → エヌイーシー
  NTT → エヌティーティー
  JDI → ジェイディーアイ
  AGC → エージーシー
  JSR → ジェイエスアール
  TDK → ティーディーケー
  NGK → エヌジーケー
  DOE → ディーオーイー（米エネルギー省の場合）
  IEA → アイイーエー
  NEDO → ニード（またはエヌイーディーオー）
  METI → メティ（またはエムイーティーアイ）

#### ルール3: 迷ったときの判断基準
- その略語を声に出して読んだとき、日本語の一般リスナーが意味を理解できるか
- 理解できない場合は、カタカナ表記 ＋ 括弧内に意味を説明する形にする
  例: 「シーエーティーエル（中国の大手電池メーカー）」
- 上記リストにない略語は、読み方を推測してカタカナ変換するか、
  「〜の略」という形で説明してから使う

#### ルール4: 数字・単位の読み方
- % → パーセント
- kWh → キロワットアワー
- GWh → ギガワットアワー
- TWh → テラワットアワー
- nm → ナノメートル
- μm → マイクロメートル
- Gbps → ギガビーピーエス
- Tbps → テラビーピーエス
- $10B → 100億ドル（日本語で金額を言い換える）

### その他の共通注意事項

- 記事にない情報を創作・追加しない
- 固有名詞（企業名・人名・製品名）は記事の表記に忠実に従う
- 一文は読み上げやすいよう40〜60字程度に収める（長すぎる文は分割する）
- 読点「、」を適切に入れ、自然な間合いで読み上げられるようにする
- 感嘆符「！」は使っても良いが、1発言に多くて2個まで
